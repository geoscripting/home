Vectorization: Pandas and GeoPandas¶

Pandas is based on numpy, therefore it provides vectorized computation as well.

→ Pandas User Guide: Accelerated Operations

1. Vectorization in Pandas¶

To take a closer look at vectorized computation using Pandas, we will take a look at Sofia Heisler’s repositoryPyCon 2017: Optimizing Pandas Code for Performance

This repo contains the material of her talk which she gave at the PyCon Conference 2017. Her talk is available on YouTube and she also wrote a blog post about it.

What we will do¶

Confernece talks and tutorials on GitHub are a great way to stay up-to-date with current developments in the scientific Python world and great resource to learn. Therefore, we will:

  1. Fork her GitHub Repository, so that we have our own copy of it.

  2. Clone our copied GitHub Repository to our computer.

  3. Work through the notebook to learn about Pandas.

Remeber that the package Pandas is built on Numpy. So every Pandas.DataFrame is basically a numpy.array with additional row and column anmes.

Intuitively you might want to use a for loop to iterate over the rows. Something like this:

[21]:

%%timeit
for i, d in df.iterrows():
    d["population"] / d["area"]

1.51 ms ± 54.5 µs per loop (mean ± std. dev. of 7 runs, 1000 loops each)

This however is a bad idea, because you are not making use of the built-in vectorization using the universal functions provided by numpy:

[22]:

%%timeit
df["population"] / df["area"]

129 µs ± 6.65 µs per loop (mean ± std. dev. of 7 runs, 10000 loops each)

Using the vectorized calculation, the execution is 10 times faster!

DataFrame.apply() or Series.map() methods¶

[24]:

import numpy as np

[25]:

df.loc[1, "population"] = np.nan

[ ]:

df.replace({np.nan: })

[29]:

%%timeit
df.apply(lambda x: x["population"] / x["area"], axis=1)

1.12 ms ± 155 µs per loop (mean ± std. dev. of 7 runs, 1000 loops each)

[62]:

%%timeit
df.apply(lambda x: 0 if np.isnan(x["pop"]) else x["pop"] * x["area"], axis=1)

1.27 ms ± 70.3 µs per loop (mean ± std. dev. of 7 runs, 1000 loops each)

However there is still an overhead involved when using pandas.DataFrames compared to numpy.array, since the metadata (e.g. index, columns, dtype) of the data frame have to be stored as well. When setting raw=True the apply as applied to a numpy array instead of the series. This means the column names are gone but it is a lot faster.

[67]:

%%timeit
df.apply(lambda x: -99 if np.isnan(x[0]) else x[0] * x[1], axis=1, raw=True)

1.29 ms ± 13.7 µs per loop (mean ± std. dev. of 7 runs, 1000 loops each)

2. Vectorization in GeoPandas¶

GeoPandas is great … but still a bit slow¶

Geopandas make spatial analysis in Python a lot easier, but it has still got a bottleneck. For geometric opertions it relies on shapely, which - as we have seen - is not the fastest. In addition, operations along a series of shapely objects cannot be vectorized in Python. Do you know why?

Answer:

But the GeoPandas developers also found a solution for this problem: Yet another package - PyGEOS¶

The PyGEOS packages allows vectorized geometric calculations based on the C library GEOS.

→ Take a look at the PyGEOS User Guide

PyGEOS integration in GeoPandas¶

The support of PyGEOS in GeoPandas is already partly implemented. So if pygeos is installed in your python environment you can enable the pygeos support. But beware, it is still in development phase!

→ Take a loot at the changes

→ Follow the development on GitHub: * GeoPandas performance: optimizing vectorized operations * Integrating pygeos in GeoPandas for vectorized array operations
