Writing good scientific Code¶

for Readability, Maintainability and Reusability¶

What is the difference between a Software Developer and a Scientific Programmer?¶

What are their respective goals?

Example: complicated list comprehension vs. explicit loop¶

Example: find the error in a complicated list comprehension or a foor loop¶

In this exercise we will try to maximize the readability, maintainability and reusability of our code. Forget about efficiency for a moment.

Remember: Write code for people, not computers!¶

So don’t just write down instructions, instead:

Write code that reveals its meaning!¶

How to write readable, maintainable and reusable code?¶

As a general tipp, stick to the Style Guide for Python (PEP8)

→ pylint my_program.py will check your code for compliance with PEP8

There are no strict rules and it depends on your preferences

Don’t mix different forms of formatting, or naming styles (CamelCase vs. snake_case)

[1]:

elapsedTimeInDays = 30 # CamelCase
elapsed_time_in_days = 30 # snake_case

Use noun or noun phrase names like Customer, WikiPage, Account, and AddressParser. (CamelCase) A class name should not be a verb.

Use verb or verb phrase names like post_payment, delete_page, or save.

In Python: Function names should be lowercase with words separated by underscores as necessary to improve readability. (snake_case)

Split your code into Functions¶

Remeber confusing functions like this? Nobody wants to see this when they are having a first glance at a program. Wrap them into functions that you call form within the main program.

[30]:

def does_something_complicated(seq, sub):
    target = np.dot(sub, sub)
    candidates = np.where(np.correlate(seq, sub, mode='valid') == target)[0]
    check = candidates[:, np.newaxis] + np.arange(len(sub))
    mask = np.all((np.take(seq, check) == sub), axis=-1)
    return candidates[mask]

Don’t repeat yourself!¶

Wrap operations that are done several times in a function. Saves you time and makes your code more robust, because you are less likely to introduce inconsistencies, when make changes.

[26]:

import numpy as np
band1 = np.array([1.,4.,6., -99.])
band2 = np.array([4., -99., 5., -99])

band1 = np.where(band1 == -99, np.nan, band1)
band2 = np.where(band2 == -99, np.nan, band1)

Split your code into Packages and Modules¶

Module: Python file that contain functions and classes

package: A folder that contains modules

Your project folder may look something like this:¶

my_python_project

     main.py

     _init_.py   ← empty file, but needed for import in Python

     project_utils.py    ← this is a module

     general_utils.py   ← this is a module

     other_utils   ← this is a package

        __init__.py   ← empty file, but needed for import in Python

         some_functions_and_classes.py   ← this is a module

         some_other_functions.py   ← this is a module

Main Program File¶

Your main program file shows the general structure of the program (it contains the “plot” of your program)

Which objects and processes are involved, what is happening?

This is the file that is being executed to start your program.

You can use a normal Python file or a Jupyter Notebook for this.

Modularisation¶

Store functions and classes in modules and packages.

To increase the reusability of your code store project specific and non-project specific functions in two different modules:

  • project_utils.py: project specific functions only used in this project

  • general_utils.py: general purpose functions that you also use in other projects

Use Pseudo code to design your program¶

When start out writing a program, you may write some pseudo code first to design the general structure of your program. Here are some tipps.

Do’s¶

  • Use control structures (if … else, for, …)

  • Use naming conventions

  • Use indentation

  • Keep it simple and consise

Don’t¶

  • Don’t make the pseudo code abstract

  • Don’t be too generalized

  • Don’t describe the implementation, describe the meaning of the code parts

Example:¶

if “1”

print response

    "I am case 1"

if “2”

print response

    "I am case 2"

class Walker():

    def __init__(self, thresh_for_walking_down, thresh_for_walking_up, start_position):
        self.__WALK_UP = 1
        self.__STAY = 0
        self.__WALK_DOWN = -1

        self.thresh_for_walking_down = thresh_for_walking_down
        self.thresh_for_walking_up = thresh_for_walking_up
        self.start_position = start_position
        self.positions = [start_position]
        self.probabilities = self.generate_probabilities()

    def take_a_step(self):
        self.get_prob_for_next_step()
        if self.can_walk_up():
            self.add_new_position(self.__WALK_UP)
        elif self.can_walk_down():
            self.add_new_position(self.__WALK_DOWN)
        else:
            self.add_new_position(self.__STAY)

    def add_new_position(self, next_step):
        self.positions.append(self.last_position() + next_step)

    def can_walk_up(self):
        return self.next_prob > self.thresh_for_walking_up and self.last_position() < 4

    def can_walk_down(self):
        return self.next_prob < self.thresh_for_walking_down and self.last_position() > 1

    def generate_probabilities(self):
        return np.random.random(1000)

    def get_prob_for_next_step(self):
        self.next_prob = np.random.random(1)

    def last_position(self):
        return self.positions[-1]

Below is the code for a 2-Dimensional Random Walker. The instructions are there, but the story does not become apparent in the code. You can do a better job than that! Make it more readable, maintainable and reusable.

# Python code for 2D random walk.
import numpy
import pylab
import random

# defining the number of steps
n = 100000

#creating two array for containing x and y coordinate
#of size equals to the number of size and filled up with 0's
x = numpy.zeros(n)
y = numpy.zeros(n)

# filling the coordinates with random variables
for i in range(1, n):
    val = random.randint(1, 4)
    if val == 1:
        x[i] = x[i - 1] + 1
        y[i] = y[i - 1]
    elif val == 2:
        x[i] = x[i - 1] - 1
        y[i] = y[i - 1]
    elif val == 3:
        x[i] = x[i - 1]
        y[i] = y[i - 1] + 1
    else:
        x[i] = x[i - 1]
        y[i] = y[i - 1] - 1


# plotting stuff:
pylab.title("Random Walk ($n = " + str(n) + "$ steps)")
pylab.plot(x, y)
pylab.savefig("rand_walk"+str(n)+".png",bbox_inches="tight",dpi=600)
pylab.show()
